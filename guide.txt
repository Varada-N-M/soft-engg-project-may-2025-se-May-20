## 1. APPLICATION SETUP

### How to Run the Application:
1. Open terminal in project root directory
2. Make script executable (first time only):
   ```
   chmod +x run_all.sh
   ```
3. Run the application:
   ```
   ./run_all.sh
   ```

### Access URLs:
- Frontend Application: http://localhost:5173
- Backend API: http://127.0.0.1:5000
- API Documentation: http://127.0.0.1:5000/static/swagger.html

### Database Setup:
- The application uses SQLite database
- Run `python backend/migrate.py` to seed the database with test data
- Database file: `backend/database.db`

3. **Manage Teachers**

## 6. TROUBLESHOOTING

### Common Issues:

1. **Cannot Access Application**
   - Ensure both backend and frontend servers are running
   - Check if ports 5000 and 5173 are available
   - Verify run_all.sh script has execute permissions

2. **Login Issues**
   - Verify credentials match those in migrate.py
   - Check if database has been seeded properly
   - Clear browser localStorage if authentication is stuck

3. **Database Issues**
   - Run `python backend/migrate.py` to recreate database
   - Ensure database.db file exists in backend directory
   - Check if all required dependencies are installed

4. **Route Access Issues**
   - Authentication is required for most routes
   - Users can only access routes for their role type
   - Check if user is logged in with correct role

5. **CORS Issues**
   - Backend is configured for localhost:5173
   - If using different port, update CORS settings in main.py

### Navigation Tips:
- Use browser back button carefully with authentication
- Logout properly to clear session data
- Each user type has specific home page after login
- Protected routes automatically redirect to appropriate login page
